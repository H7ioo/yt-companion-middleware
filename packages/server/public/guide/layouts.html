16 Stream Deck layouts

A layout is a page of keys you can run a whole service from without thinking. These three cover the cases this app was built for.

The keys below are simulations — the same colours and words the Companion module puts on a real deck, painted from a mock state you control. Press a preset key to make it the active one; use the panel to change what the app is doing underneath.

Health

Read the lamps, not the log. The health key is the one to keep on every page: it recolours itself — green healthy, amber degraded, slate offline, red auth error — so a stream that has quietly stopped updating announces itself from across the room. What each state means.

Why these three

  • Preset wall — the default. Every preset is one press away, and the bottom row is state: on air, busy, undo, health. Nothing on this page can lose you a broadcast, which is the point.
  • One-service page — for a volunteer who runs one thing and shouldn't have to choose. The Arabic title renders as an image key (Companion's fonts can't draw Arabic text).
  • Tech desk — the two refreshes are separate keys and are never given the same name: Refresh from YouTube re-reads the broadcast, Refresh lists re-pulls the presets and category pickers.

Every key here comes from the module's Presets tab pre-wired — drag, drop, done. See Presets (drag-drop).
