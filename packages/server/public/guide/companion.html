10 Companion · install & connect

This repo ships a dedicated Companion module (companion-module/). It holds a live WebSocket to the middleware — instant push, zero YouTube quota — and puts the middleware's presets, actions, feedbacks and Arabic-safe title/slug images on your keys. Prefer it over the Generic HTTP module.

Install the module (Companion 4+)

Modules are plugins you install independently. Build a package once with a single command from the repo root — it installs the module's deps and bundles the .tgz, no cd, no restart: 

npm run companion:package   # → companion-module/yt-companion-middleware-<version>.tgz

In Companion open Modules → Import module package and pick that .tgz. For development instead point Settings → Developer modules path at a folder holding (or symlinking) companion-module/ (run npm run companion:install once first), then restart Companion — edits hot-reload.

Add the connection

  1. Connections → Add connection, search yt-companion-middleware, add it.
  2. Set Middleware base URL to http://<APP_IP>:8080 (same host as the dashboard). HTTPS is fine — the module derives wss:// itself. Save.

There is no token to set — the middleware is unauthenticated (LAN-only), so the module has no Bearer token field.

Watch the status pill: it goes Connecting → OK once the WebSocket is up. Stuck on Connection failure means a wrong/unreachable URL or a rejected token — fix it and it reconnects on its own. Confirm live: set any key's text to $(ytmeta:display_label).

11 Companion · presets (drag-drop)

The module publishes ready-made buttons in Companion's Presets tab — the answer to "I want a key that applies a preset and labels itself." Drag one onto a key and it arrives already labelled, already wired, already lit when active. No JSON, no manual action.

Category · Apply preset

One button per middleware preset, generated from your dashboard. Each drops with:

  • its slug as the button text (Latin-safe label);
  • the Apply preset action already bound to that preset's id;
  • the Active preset is… feedback already pointed at the same preset, so the key lights violet whenever that preset is the one on air.

Added or renamed presets in the dashboard? The Presets tab is snapshotted when the connection starts. Run the Refresh lists action once (or re-save the connection) and the new preset buttons appear.

Presets are just a starting point — every dropped button is fully editable afterwards (add a second feedback, swap the text for an image, change colors). Presets are authored by the module; you can't create new ones from the UI, but you can export your edited buttons as a custom library.

12 Companion · every action, with suggested usage

All actions POST to the middleware's /api/action/* bus; state comes back over the WebSocket, so you never add a manual refresh after one. Add them under a key's Press (or any step). Fields marked supports $(…) accept Companion variables.

ActionOptionsSuggested usage
Apply preset Preset (dropdown); Template vars (JSON, optional, supports $(…)) The everyday "switch what's on air" key. For a fixed preset, just pick it — or drag its ready-made button from the Presets tab. For a preset with placeholders, pass a JSON object like {"lesson":"41"}; pull the value from a Companion variable if you have one. No value and no fallback? Use the redirect/fill flow (§14) so the operator types it in a browser first.
Request fill (popup on dashboard / phone push) Preset (dropdown) For presets the operator must type values into. Surfaces the fill page instead of prompting on the key (Companion can't): any open dashboard pops the fill dialog, and with ntfy configured the phone gets a tap-to-open notification. See §14.
Update live metadata Title (required, $(…)); Description; Privacy; Category; Bound stream Ad-hoc edit without a preset. Leave Privacy/Category/Bound stream on "unchanged" to keep current values; only Title is always sent. Great bound to a variable, e.g. $(internal:custom_myTitle), for a type-then-push workflow.
Privacy: toggle private ↔ public — A one-key "go public / go private" panic switch. Pair it with the Privacy toggle preset so the key label always shows the current state.
Privacy: set Status (public / unlisted / private) Three explicit keys when you want a dedicated public / unlisted / private button rather than a toggle.
Undo last change — Reverts the last metadata change. $(ytmeta:undo_label) on the key tells the operator what "undo" will revert before they press it.
Refresh from YouTube — Forces the middleware to re-pull state from YouTube. Rarely needed (push keeps you current) — use it if you changed something in YouTube Studio directly and want it reflected now.
Refresh lists — Re-pulls the dropdown choices and regenerates the Presets tab after you add/rename presets in the dashboard. Bind it to a spare key or press it from the button editor.
API master switch (kill switch): set / toggle set: API (enabled / disabled); toggle: — Turns the middleware's master switch on/off. While off it makes no YouTube calls and rejects actions — a one-key way to stop quota burn on an idle service. Pair the toggle with the API disabled feedback so the key shows on/off; the drop-in API kill switch preset already does.

A key can't open a browser — Companion has no "Open URL" action. Filling a template-var preset goes through the Request fill action instead (see §14); opening the dashboard itself is a bookmark on the device in your hands. A rejected action (token required, empty title, etc.) is logged in Companion's Log tab as a rejected/failed line, not raised on the key.

Arabic (non-Latin) titles on a key

Companion's bundled fonts have no Arabic glyphs and do no RTL shaping, so an Arabic live_title bound as plain key text renders as boxes (tofu). This module is the fix: the middleware pre-renders the label and the full title to PNGs with a bundled Arabic font (shaped + joined), and the module turns them into the two image feedbacks above — Image: button label (slug) and Image: full live title.

Two-step toggle. A full title rarely fits a key, so make it a two-step button: step 1 carries the slug image feedback, step 2 the title image — one press flips between the short label and the full title.

Latin-safe fallback. If a preset's slug is Latin, you can skip the image entirely and just bind the key text to $(ytmeta:display_label).

Added a button after the connection was already up and it shows no value yet? The WebSocket re-sends state on its heartbeat, or run the Refresh from YouTube action / re-save the connection to force a fresh frame now. See companion-module/README.md for full install and packaging notes.
